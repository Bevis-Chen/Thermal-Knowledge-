專案開發到工廠生產之流程:

開案前  
  - 發想 / study  (高層...)
  - OEM(Original Equipment Manufacturer)
    ODM(Original Design Manufacturer)
    OBM(Own Brand Manufacturer)
    ref: https://vocus.cc/article/646c1cd7fd89780001b2a2b1
    ref: https://zhuanlan.zhihu.com/p/1962083772678772460
    ref: https://seller.alibaba.com/tw/blogs/oem-odm-obm-differences
    OEM >> 完全客戶品牌
    OBM >> 白牌/自由品牌，企業自主開發但未必自己生產.
    ODM >> OBM商品"貼牌"變成客戶商品. 
  - 承上. ODM/OBM專案的開發最初、沿用機構外型以及搭配最新的晶片還有外接卡等.
  - OEM需遵循客戶條件. 
  - 定SPEC >> 初步規劃要塞多少東西或有多少發熱元 (高層或主管)
  - 有個 整機 placement 的構想: thermal architecture >> 氣冷或是那些需要水冷
開案    
  - 劃定 schedule >> 劃定EVT以及DVT 多少周
  - 準備以及初步 跑模擬以及輸出結果 : 
    比對當初開案前發想的結果還差多少. 
    檢驗是否此專案構想可不可行.
    在EVT具體進一步驗證. 
  - mock-up 機 >>
    - 有個不開機系統裝CPU-TTV 實測CPU可以到多少溫度、
    - 可以推估跟設計HEATSINK要設計多少?
    - 風扇選用
EVT 工程驗證測試 "第一個真正可以驗證設計假設的時候。"
  - 驗證 跟 開案構想以及模擬的結果差多少
    - 大概率要debug 能不能開機>>系統firmware、硬件CPU/DIMM/SSD/GPU/CARD等等進入實測支援程度
  - 最好測個整機風洞>> 可以知道系統有多少流量.
  - 風扇跟heatsink 的確認 : 單體風洞測試 一下
  - 導風罩或其他solution
DVT Design Verification Test 設計驗證測試
  - 理論上 接近成品 >> 必須要可以開機了
  - 因為 EVT可能設計導風罩、或者有EVT時候有別於開案的設計變更. 
  - 這個階段做總體測試
    - idle / full stress
    - 25/ 30 / 35 環溫
    - fan duty >> max 檢驗可以散熱多少 / auto 檢驗能不能support 
  - 我還是不懂TEST PLAN
  - 應該是劃定測什麼跟測多久
  - 再測個風洞 >> 看最後吹出多少流量跟阻抗. 跟熱測試結果對比.
PVT production validation test
  - 驗證製成上的驗證測試 >> 這個我沒經歷過. 應該是試產?
  - 如果DVT有什麼重大設計變更(比如DVT的solution、或是電路線路甚至晶片有特殊變化/更高瓦數)
  - 單機測試比較少. 
MP mass production
  - 其實我沒有經歷過XDD
  - 其實GPT有說這個階段要看大量生產: 
    總不能單機測試CPU可以支援500W/88度、大量生產測試(柯南測試軟體?/或其他用於大量生產的軟體 / BIOS寫的TOOL)
    整櫃的系統測試發現過不了(蠻有可能的、互相搶風>> 所以我會以此認定thermal buffer盡量抓5度以上 / 1~3度的裕度就算PASS的真是太勇敢了)
